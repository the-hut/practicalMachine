Loaded the data again but this time cleaned up all the NA values and then split it into the training and testing sets with a 70% ratio.

data <- read.csv("pml-training.csv", na.strings = c("#DIV/0!", "NA"), row.names = 1)

set.seed(1122)
inTrain <- createDataPartition(y = data$classe, p = 0.7, list = FALSE)
training <- data[inTrain, ]
testing <- data[-inTrain, ]

Looked to see if we could find any columns with a high frequency of NA values and plotted the results.

qplot(colSums(is.na(training)), xlab = "Number of NAs in Column", ylab = "Number of Columns",

Also noted that the first six columns have no meaning with regards to the target variable. Two steps were thus taken: 1 Removed the first 6 columns. 2 Removed all columns with an NA frequency of higher than 90%. 3 Applied these changes to the training dataset as well.

The resultant plot shows the resultant number of columns and their NA frequency.

training <- training[, -c(1:6)]
training <- training[, colSums(is.na(training)) < nrow(training)/100 * 90]

testing <- testing[, -c(1:6)]
testing <- testing[, colSums(is.na(testing)) < nrow(testing)/100 * 90]
